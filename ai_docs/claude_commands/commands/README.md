## Usage Tips

1. Always start with `@context` to load project context
2. **IMPORTANT**: Use `@lore_conventions` to ensure proper naming and terminology
3. Use `@review_existing_code` before implementing new features
4. Use specific implementation guides for detailed component guidance
5. Refer to `@build_project` for the overall implementation plan and dependencies
6. Run `@verify_build` before submitting code to ensure quality
7. Check `@module_structure` to maintain consistent organization

## Implementation Workflow

For implementing a new feature:

1. `@context` - Load project context
2. `@lore_conventions` - Review and apply Guild's naming conventions
3. `@review_existing_code` - Check what exists already
4. `@tdd` - Set up tests first
5. `[implementation command]` - Implement the feature
6. `@ensure_tests` - Verify test coverage
7. `@verify_build` - Check build and code quality
8. `@go_best_practices` - Review for Go standards compliance

## Objective System Workflow

For implementing the objective system:

1. `@context` - Load project context
2. `@lore_conventions` - Review and apply Guild's naming conventions
3. `@review_existing_code` - Check what exists already
4. `@implement_prompt_system` - Implement prompt management
5. `@implement_generator_package` - Implement generator functionality
6. `@implement_objective_system` - Implement core objective functionality
7. `@implement_objective_ui` - Implement Bubble Tea UI
8. `@verify_build` - Check build and code quality

For other components, follow similar workflow patterns, always starting with context and reviewing existing code.
