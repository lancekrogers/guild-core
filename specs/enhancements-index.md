# 🌱 Planned Enhancements Index

This index tracks advanced or optional features that extend Guild beyond the core system. Each of these is implemented in a self-contained spec for future development.

## 🧠 Enhancement Topics

| Feature                            | Status  | Spec File              | Summary                                                                    |
| ---------------------------------- | ------- | ---------------------- | -------------------------------------------------------------------------- |
| Model routing by task complexity   | Planned | `model-routing.md`     | Dynamically choose model per task based on complexity, role, or cost.      |
| Versioned specs + live updates     | Planned | `spec-versioning.md`   | Add git-based or metadata-based versioning to objective files.             |
| Permissioning & sandboxing         | Planned | `agent-permissions.md` | Restrict agents/tools to specific folders, commands, or data domains.      |
| Retry & failure handling           | Planned | `task-resilience.md`   | Define how failed tasks are retried, escalated, or logged.                 |
| Agent personality / role templates | Planned | `agent-templates.md`   | Predefined behavior and tone templates for roles (e.g., Editor, Reviewer). |

Each file describes goals, spec format, and how the feature would integrate into the Guild runtime.

These are not yet implemented but are ready for community or solo development work.
